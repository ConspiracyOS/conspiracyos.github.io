# Contributing

We welcome contributions from anyone who thinks agent infrastructure
should be inspectable, auditable, and self-hosted.

## Code of Conduct

Be direct. Be respectful. Assume good faith.
Technical precision matters more than tone policing.

## Where to Start

- [Issues](https://github.com/ConspiracyOS/conctl/issues) on the conctl repo
- [Discussions](https://github.com/orgs/ConspiracyOS/discussions) for questions and ideas
- Look for the `good-first-issue` label

## What We Need

- Bug reports with reproduction steps
- Documentation improvements
- New agent skills and contract definitions
- Platform support (currently arm64/amd64 Linux)

## Architecture

Significant design decisions are documented internally. If you are
proposing a large change, open an issue first. We can share the
relevant context and save you from building in the wrong direction.

The project is split across several repos. The monorepo structure is
described in the [GitHub org](https://github.com/ConspiracyOS).

## Pull Requests

1. Fork the repository
2. Create a feature branch
3. Write tests where applicable
4. Open a PR with a clear description
5. Address review feedback
